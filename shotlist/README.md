# shotlist/ — the site-visit film portal

Auto-generated by `scripts/gen-shotlist.mjs` (command: `/shotlist`). Phase ① of
the site-visit video workstream — see [`../NAC-SITE-VISIT-VIDEO.md`](../NAC-SITE-VISIT-VIDEO.md).

## The portal — `shotlist/index.html`

**One page, bookmark it.** Every listing is inlined, so it works **offline
on-site** (no signal needed after it loads). Three stages:

1. **Prepare / Browse** — search + country filter over every property; tap ★ to
   add a stop to *My Trip* (saved on your device).
2. **Trip** — your prepared stops with total shots + footage; tap to open.
3. **Read** — the on-site screen for one property: bilingual VI/EN toggle,
   progress ring, tap each shot to tick it off (ticks saved per device). ★ shots
   are hero / banner-video candidates.

Live URL (once merged to `main`):
`https://rayvtt.github.io/Nac-Property-Hub-Listing-PDP/shotlist/`

The portal is **regenerated from every `properties/*.html`** on each run, so new
listings appear automatically. Do not hand-edit — rerun `/shotlist`.

## Per-listing markdown (optional)

`node scripts/gen-shotlist.mjs <slug>` also writes `shotlist/<slug>.md` — the
same script as a table, for pasting into Notion. `--city <name>` writes one per
matching listing. Filming convention printed in every script: shoot each shot as
`<slug>__<ID>.mp4` so the upload step can auto-route the clips.
